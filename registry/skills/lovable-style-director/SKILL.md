# Lovable Style Director

## Objetivo

Elevar a direção visual de landing pages, sites e aplicações SaaS para um nível premium, com foco em composição, ritmo visual e experiência completa de produto. Evitar o visual genérico de template.

## Quando usar

- Landing page que parece "template Tailwind genérico"
- Projeto SaaS que precisa de aparência de produto completo
- Quando o design atual não comunica qualidade
- Refatoração visual de projetos existentes
- Criação de componentes com identidade visual forte

## Quando não usar

- Projetos puramente técnicos sem interface
- Quando já existe design system consolidado e aprovado
- Tarefas de correção de bug visual pontual

## Entradas esperadas

- Descrição do projeto/produto
- Público-alvo
- Referências visuais (opcional)
- Marca (se existir)

## Processo

1. Analisar o propósito do projeto e público
2. Definir atmosfera visual (Premium / Tech / Clean / Bold / Minimal)
3. Criar ou refinar paleta de cores
4. Definir tipografia e escala
5. Planejar grid e composição
6. Desenhar hierarquia visual das seções
7. Criar componentes com personalidade
8. Garantir ritmo visual (alternância, respiro, densidade)
9. Revisar consistência entre seções

## Checklist

- Paleta com 3-5 cores definidas
- Escala tipográfica clara
- Grid consistente
- Componentes com personalidade (não genéricos)
- Ritmo visual entre seções
- Consistência de espaçamento
- Hierarquia visual correta
- Hero impactante
- CTA visível e bem posicionado
- Responsividade preserva a qualidade visual

## Formato de saída

```md

[Premium / Tech / Clean / Bold / Minimal] — [descrição]

## Paleta
- Primary: #HEX
- Secondary: #HEX
- Accent: #HEX
- Background: #HEX
- Text: #HEX

## Tipografia
- Headings: [fonte]
- Body: [fonte]

## Grid
[Descrição do grid e breakpoints]

## Componentes-chave
[Estilo de hero, cards, CTAs, navegação]

## Ritmo visual
[Como as seções se alternam visualmente]
```

## Regras de qualidade

- Nunca usar paleta padrão de framework
- Nunca usar tipografia sistema sem customização
- Sempre definir tokens de design
- Sempre pensar mobile-first
- Evitar gradientes genéricos
- Evitar sombras excessivas
- Evitar ícones padrão sem contexto

## Não fazer

- Não copiar templates prontos
- Não usar cores sem paleta definida
- Não ignorar a identidade da marca
- Não criar layout sem hierarquia
- Não esquecer estados (hover, focus, active)

## Exemplo de uso

**Projeto**: SaaS de agendamento para clínicas
**Atmosfera**: Clean + Profissional + Acolhedor
**Paleta**: Azul sereno (#2563EB), Branco (#FFFFFF), Cinza quente (#F5F5F4), Verde confirmação (#16A34A)
**Tipografia**: Inter (headings + body)
**Grid**: 12 colunas, max-width 1280px
